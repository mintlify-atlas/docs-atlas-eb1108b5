---
title: "Peer Class — Connected Peer State and Data Access"
description: "Query state, transfer rates, and identity of active peers, and manage PeerList, ConnectionList, PeerInfo, and related supporting types."
---

`Peer` is a read-mostly handle to a single active peer connection. libTorrent may invalidate a `Peer*` during any call to `torrent::work()` — if you store peers across work calls, listen to `ConnectionList::signal_disconnected()` and remove invalidated entries. You cannot construct a `Peer` directly; libTorrent hands you pointers through `ConnectionList` or `ConnectionList::signal_connected()`.

```cpp
#include <torrent/peer/peer.h>
#include <torrent/peer/connection_list.h>
```

<Warning>
  A `Peer*` becomes invalid after the peer disconnects. Always handle `signal_disconnected` to purge stale pointers from your data structures before the next `work()` call.
</Warning>

---

## State queries

```cpp
namespace torrent {

class Peer {
public:
  bool is_incoming()  const;
  bool is_encrypted() const;
  bool is_obfuscated() const;
};

}
```

| Method | Returns | Description |
|---|---|---|
| `is_incoming() const` | `bool` | `true` when the remote peer connected to us (inbound connection). |
| `is_encrypted() const` | `bool` | `true` when the connection uses MSE/PE stream encryption. |
| `is_obfuscated() const` | `bool` | `true` when header obfuscation is active on this connection. |

## Choke state

libTorrent tracks two independent choke directions: **up** (what we are sending to the peer) and **down** (what the peer is sending to us).

```cpp
namespace torrent {

class Peer {
public:
  bool is_up_choked()         const;  // we are choking them
  bool is_up_interested()     const;  // they are interested in us

  bool is_down_choked()       const;  // they are choking us
  bool is_down_choked_limited() const;
  bool is_down_queued()       const;
  bool is_down_interested()   const;  // we are interested in them
};

}
```

| Method | Returns | Description |
|---|---|---|
| `is_up_choked() const` | `bool` | `true` when we are currently choking this peer (not sending pieces to them). |
| `is_up_interested() const` | `bool` | `true` when the peer has signalled interest in downloading from us. |
| `is_down_choked() const` | `bool` | `true` when the peer is choking us (not sending pieces to us). |
| `is_down_choked_limited() const` | `bool` | `true` when the unchoke is being held back by the resource manager's slot limit. |
| `is_down_queued() const` | `bool` | `true` when this peer is in the unchoke queue but not yet unchoked. |
| `is_down_interested() const` | `bool` | `true` when we are interested in downloading from this peer. |

## Behavior control

```cpp
namespace torrent {

class Peer {
public:
  bool is_snubbed() const;
  void set_snubbed(bool v);

  bool is_banned() const;
  void set_banned(bool v);
};

}
```

| Method | Returns | Description |
|---|---|---|
| `is_snubbed() const` | `bool` | `true` when this peer is snubbed (we stop requesting blocks from them temporarily). |
| `set_snubbed(bool v)` | `void` | Marks or clears the snubbed flag. Set `true` when the peer is too slow to serve blocks. |
| `is_banned() const` | `bool` | `true` when this peer is banned from reconnecting. |
| `set_banned(bool v)` | `void` | Marks or clears the banned flag. A banned peer will not be reconnected to by the library. |

<Tip>
  Snubbing a peer does not disconnect it — libTorrent stops sending block requests to the peer but keeps the connection open and may still upload to them.
</Tip>

## Identity

```cpp
namespace torrent {

class Peer {
public:
  const HashString& id()        const;
  const char*       options()   const;
  const sockaddr*   address()   const;
  const PeerInfo*   peer_info() const;
};

}
```

| Method | Returns | Description |
|---|---|---|
| `id() const` | `const HashString&` | The 20-byte BitTorrent peer ID exchanged during the handshake. |
| `options() const` | `const char*` | Pointer to the 8-byte extension flags from the handshake. Use to test protocol support (e.g. DHT, extensions). |
| `address() const` | `const sockaddr*` | Remote socket address. Cast to `sockaddr_in` or `sockaddr_in6` depending on `sa_family`. |
| `peer_info() const` | `const PeerInfo*` | Pointer to the `PeerInfo` record for this peer. See the [PeerInfo section](#peerinfo) below. |

## Transfer rates

```cpp
namespace torrent {

class Peer {
public:
  const Rate* down_rate()  const;
  const Rate* up_rate()    const;
  const Rate* peer_rate()  const;
};

}
```

| Method | Returns | Description |
|---|---|---|
| `down_rate() const` | `const Rate*` | Bytes per second being received from this peer. |
| `up_rate() const` | `const Rate*` | Bytes per second being sent to this peer. |
| `peer_rate() const` | `const Rate*` | The download rate that the peer has reported for itself in extension messages. |

Each `Rate` pointer remains valid as long as the `Peer` is valid. See [Rate](/api/throttle#rate-class) for the full interface.

## Data and queues

```cpp
namespace torrent {

class Peer {
public:
  const Bitfield*      bitfield()            const;
  const BlockTransfer* transfer()            const;

  uint32_t             incoming_queue_size() const;
  uint32_t             outgoing_queue_size() const;
  uint32_t             chunks_done()         const;
  uint32_t             failed_counter()      const;
};

}
```

| Method | Returns | Description |
|---|---|---|
| `bitfield() const` | `const Bitfield*` | The set of pieces this peer has announced. May be sparse or incomplete early in a connection. |
| `transfer() const` | `const BlockTransfer*` | The block currently being transferred, or `nullptr` if idle. |
| `incoming_queue_size() const` | `uint32_t` | Number of block requests we have outstanding to this peer. |
| `outgoing_queue_size() const` | `uint32_t` | Number of block requests the peer has outstanding to us. |
| `chunks_done() const` | `uint32_t` | Number of pieces this peer has completed since the connection was established. |
| `failed_counter() const` | `uint32_t` | Number of failed or timed-out block requests for this connection. |

## PeerInfo

`PeerInfo` stores persistent metadata about a peer that survives reconnections. You access it from `Peer::peer_info()` or directly from `PeerList`.

```cpp
#include <torrent/peer/peer_info.h>

namespace torrent {

class PeerInfo {
public:
  // Flags
  static constexpr int flag_connected = (1 << 0);
  static constexpr int flag_incoming  = (1 << 1);
  static constexpr int flag_handshake = (1 << 2);
  static constexpr int flag_blocked   = (1 << 3);
  static constexpr int flag_restart   = (1 << 4);
  static constexpr int flag_unwanted  = (1 << 5);
  static constexpr int flag_preferred = (1 << 6);

  bool             is_connected()  const;
  bool             is_incoming()   const;
  bool             is_handshake()  const;
  bool             is_blocked()    const;
  bool             is_restart()    const;
  bool             is_unwanted()   const;
  bool             is_preferred()  const;

  int              flags()         const;

  const HashString& id()           const;
  const char*       id_hex()       const;
  const ClientInfo& client_info()  const;
  const char*       options()      const;
  const sockaddr*   socket_address() const;

  uint16_t          listen_port()  const;

  uint32_t          failed_counter()    const;
  void              set_failed_counter(uint32_t c);

  uint32_t          transfer_counter()  const;
  uint32_t          last_connection()   const;
  void              set_last_connection(uint32_t tvsec);
  uint32_t          last_handshake()    const;
  void              set_last_handshake(uint32_t tvsec);

  bool              supports_dht()        const;
  bool              supports_extensions() const;
};

}
```

<AccordionGroup>
  <Accordion title="Flag constants">
    | Constant | Meaning |
    |---|---|
    | `flag_connected` | Peer has an active connection. |
    | `flag_incoming` | The connection was initiated by the remote peer. |
    | `flag_handshake` | Handshake is in progress. |
    | `flag_blocked` | Peer is blocked from being sent to (used by initial seeding). |
    | `flag_restart` | Peer should be reconnected on the next opportunity. |
    | `flag_unwanted` | Peer is marked unwanted in the IP table. |
    | `flag_preferred` | Peer is preferred in the IP table. |
  </Accordion>
  <Accordion title="Timing fields">
    `last_connection()` and `last_handshake()` both return a Unix timestamp (`uint32_t` seconds). You can call `set_last_connection()` and `set_last_handshake()` to override them, but in normal use libTorrent maintains these values automatically.
  </Accordion>
  <Accordion title="Extension capability">
    `supports_dht()` checks bit `0` of byte `7` of the extension flags. `supports_extensions()` checks bit `4` of byte `5`. Both read the raw `options()` buffer.
  </Accordion>
</AccordionGroup>

## PeerList

`PeerList` holds all `PeerInfo` objects for a download — both connected and known-but-disconnected peers. Access it via `Download::peer_list()`.

```cpp
#include <torrent/peer/peer_list.h>

namespace torrent {

class PeerList {
public:
  // Insert flags
  static constexpr int address_available       = (1 << 0);

  // Connect flags
  static constexpr int connect_incoming        = (1 << 0);
  static constexpr int connect_keep_handshakes = (1 << 1);
  static constexpr int connect_filter_recent   = (1 << 2);

  // Disconnect flags
  static constexpr int disconnect_available    = (1 << 0);
  static constexpr int disconnect_quick        = (1 << 1);
  static constexpr int disconnect_unwanted     = (1 << 2);
  static constexpr int disconnect_set_time     = (1 << 3);

  // Cull flags
  static constexpr int cull_old                = (1 << 0);
  static constexpr int cull_keep_interesting   = (1 << 1);

  PeerInfo* insert_address(const sockaddr* address, int flags);

  static ipv4_table* ipv4_filter();

  const auto& available_list();
  uint32_t    available_list_size() const;

  uint32_t    cull_peers(int flags);

  // Iterators (const only — PeerList is not directly mutable by callers)
  const_iterator         begin()  const;
  const_iterator         end()    const;
  const_reverse_iterator rbegin() const;
  const_reverse_iterator rend()   const;

  size_t size()  const;
  bool   empty() const;
};

}
```

| Method | Returns | Description |
|---|---|---|
| `insert_address(const sockaddr* address, int flags)` | `PeerInfo*` | Creates or retrieves a `PeerInfo` for `address`. Pass `address_available` to mark it as immediately connectable. |
| `ipv4_filter()` | `ipv4_table*` | Static global IPv4 address table used to flag peers as unwanted or preferred. |
| `available_list()` | `const auto&` | Reference to the list of addresses currently available for outgoing connections. |
| `available_list_size() const` | `uint32_t` | Number of peers currently in the available list. |
| `cull_peers(int flags)` | `uint32_t` | Removes stale or unwanted peer records. Returns the number removed. Pass `cull_old` and/or `cull_keep_interesting`. |

## ConnectionList

`ConnectionList` holds `Peer*` objects for all currently active (handshaked) connections of a single download. Access it via `Download::connection_list()`.

```cpp
#include <torrent/peer/connection_list.h>

namespace torrent {

class ConnectionList {
public:
  using size_type        = uint32_t;
  using slot_peer_type   = std::function<void(Peer*)>;
  using signal_peer_type = std::list<slot_peer_type>;

  // Disconnect flags
  static constexpr int disconnect_available = (1 << 0);
  static constexpr int disconnect_quick     = (1 << 1);
  static constexpr int disconnect_unwanted  = (1 << 2);
  static constexpr int disconnect_delayed   = (1 << 3);

  iterator  erase(iterator pos, int flags);
  void      erase(Peer* p, int flags);
  void      erase(PeerInfo* peerInfo, int flags);

  void      erase_remaining(iterator pos, int flags);
  void      erase_seeders();

  iterator  find(const char* id);
  iterator  find(const sockaddr* sa);

  size_type min_size() const;
  void      set_min_size(size_type v);

  size_type size()     const;

  size_type max_size() const;
  void      set_max_size(size_type v);

  void      set_difference(AddressList* l);

  signal_peer_type& signal_connected();
  signal_peer_type& signal_disconnected();

  // Iterators
  iterator         begin();
  iterator         end();
  const_iterator   begin() const;
  const_iterator   end()   const;
  bool             empty() const;
};

}
```

<AccordionGroup>
  <Accordion title="Size control">
    | Method | Description |
    |---|---|
    | `min_size() / set_min_size(v)` | Lower bound for desired peer connections (default 50). The connection manager targets at least this many active peers. |
    | `max_size() / set_max_size(v)` | Upper bound; no more than this many peers may be connected at once (default 100). |
    | `size()` | Returns the number of currently active connections. |
  </Accordion>
  <Accordion title="Signals">
    Both `signal_connected()` and `signal_disconnected()` return a `std::list<std::function<void(Peer*)>>` that you push callbacks into. libTorrent calls all registered callbacks when a peer connects or disconnects.

    ```cpp
    download.connection_list()->signal_connected().push_back([](torrent::Peer* p) {
      // Called each time a new peer completes its handshake
    });

    download.connection_list()->signal_disconnected().push_back([](torrent::Peer* p) {
      // Remove p from any data structures you hold before it is freed
    });
    ```
  </Accordion>
  <Accordion title="Erase and find">
    | Method | Description |
    |---|---|
    | `erase(Peer* p, int flags)` | Disconnects and removes a specific peer. |
    | `erase(iterator pos, int flags)` | Disconnects and removes the peer at `pos`. Returns the next valid iterator. |
    | `erase(PeerInfo* peerInfo, int flags)` | Disconnects the peer identified by `peerInfo`. |
    | `erase_remaining(iterator pos, int flags)` | Disconnects all peers from `pos` to `end()`. |
    | `erase_seeders()` | Disconnects all peers that are complete seeders. |
    | `find(const char* id)` | Finds a connection by 20-byte peer ID. Returns `end()` if not found. |
    | `find(const sockaddr* sa)` | Finds a connection by socket address. Returns `end()` if not found. |
  </Accordion>
</AccordionGroup>
