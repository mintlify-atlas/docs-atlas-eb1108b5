---
title: "Download — Torrent Handle and Lifecycle Class Reference"
description: "Full reference for the Download handle: lifecycle, hash checking, tracker control, peer management, file access, statistics, and download configuration."
---

`Download` is a lightweight handle — essentially a typed pointer — to an internal `DownloadWrapper` object managed by the library. It is safe to copy, assign, and destroy freely; destroying a `Download` handle does not remove the download from the library. You obtain a `Download` handle from `torrent::download_add()` or `torrent::download_find()`, and you release the underlying resources by calling `torrent::download_remove()`. All methods listed here are declared in `<torrent/download.h>`.

<Note>
  Always check `is_valid()` before calling any method on a `Download` obtained from `torrent::download_find()`. An invalid handle has a null internal pointer, and calling any other method on it causes undefined behavior.
</Note>

## Static constants

```cpp
static constexpr uint32_t numwanted_diabled = ~uint32_t();

static constexpr int open_enable_fallocate = (1 << 0);  // 0x01

static constexpr int start_no_create       = (1 << 1);  // 0x02
static constexpr int start_keep_baseline   = (1 << 2);  // 0x04
static constexpr int start_skip_tracker    = (1 << 3);  // 0x08

static constexpr int stop_skip_tracker     = (1 << 0);  // 0x01

static constexpr int update_range_recheck  = (1 << 0);  // 0x01
static constexpr int update_range_clear    = (1 << 1);  // 0x02
```

| Constant | Value | Description |
|---|---|---|
| `numwanted_diabled` | `~uint32_t()` | Pass to tracker `set_numwant` to disable the `numwant` field in announce requests. |
| `open_enable_fallocate` | `1 << 0` | Passed to `open()`: pre-allocate disk space with `fallocate(2)` when opening files. |
| `start_no_create` | `1 << 1` | Passed to `start()`: do not create missing files; fail if any output file is absent. |
| `start_keep_baseline` | `1 << 2` | Passed to `start()`: preserve the current uploaded/completed baseline counters rather than resetting them. |
| `start_skip_tracker` | `1 << 3` | Passed to `start()`: suppress the `started` tracker announce. |
| `stop_skip_tracker` | `1 << 0` | Passed to `stop()`: suppress the `stopped` tracker announce. |
| `update_range_recheck` | `1 << 0` | Passed to `update_range()`: mark the chunk range as needing a hash recheck. |
| `update_range_clear` | `1 << 1` | Passed to `update_range()`: clear the completion bitfield for the chunk range. |

## Lifecycle

```cpp
void open(int flags = 0);
void close(int flags = 0);
void start(int flags = 0);
void stop(int flags = 0);
```

| Method | Description |
|---|---|
| `open(int flags = 0)` | Opens and prepares the download's files on disk. Pass `open_enable_fallocate` to pre-allocate disk space. The download must be open before you call `start()`. |
| `close(int flags = 0)` | Closes file handles and frees memory-mapped chunks. The download must be stopped before you close it. |
| `start(int flags = 0)` | Activates the download: contacts trackers, accepts peer connections, and begins transferring data. Combine flags with bitwise-OR. |
| `stop(int flags = 0)` | Deactivates the download. Pass `stop_skip_tracker` to avoid sending a `stopped` event to trackers — useful when you plan to resume immediately. |

<Tip>
  The typical lifecycle is: `open()` → `hash_check()` → `start()` → `stop()` → `close()`. You can omit `hash_check()` if you have reliable resume data already set via `set_bitfield()`.
</Tip>

## Hash checking

```cpp
bool hash_check(bool tryQuick);
void hash_stop();
```

| Method | Returns | Description |
|---|---|---|
| `hash_check(bool tryQuick)` | `bool` | Starts verifying piece hashes against downloaded data. When `tryQuick` is `true`, stops at the first chunk that cannot be memory-mapped and returns `true` (signaling that hash checking is done and a `hash_done` signal has been queued). Chunk ranges covered by valid resume data are skipped. Call this when `is_open()` is true and before `start()`. |
| `hash_stop()` | `void` | Aborts an in-progress hash check. Safe to call even if no check is running. |

uint64_t       bytes_done() const;

## Configuration

### Resume data and bitfield

```cpp
void set_chunks_done(uint32_t chunks_done, uint32_t chunks_wanted);
void set_bitfield(bool allSet);
void set_bitfield(const uint8_t* first, const uint8_t* last);
void update_range(int flags, uint32_t first, uint32_t last);
void sync_chunks();
```

These methods must be called when the download `is_open()`, is not yet hash-checked, and is not currently hash-checking.

| Method | Description |
|---|---|
| `set_chunks_done(uint32_t chunks_done, uint32_t chunks_wanted)` | Sets the count of completed and wanted chunks for a download that has not been opened yet. Use this to restore state from a resume file. |
| `set_bitfield(bool allSet)` | Initializes the completion bitfield to all-ones (`allSet = true`) or all-zeros. Use all-ones when you are certain the download is complete. |
| `set_bitfield(const uint8_t* first, const uint8_t* last)` | Copies resume bitfield data from the byte range `[first, last)`. The range must be exactly `ceil(num_chunks / 8)` bytes. |
| `update_range(int flags, uint32_t first, uint32_t last)` | Modifies the hash-check and completion state for chunk indices in `[first, last)`. Combine `update_range_recheck` and `update_range_clear` with bitwise-OR. |
| `sync_chunks()` | Forces all dirty chunks to be flushed to disk. Call before saving resume data to ensure consistency. |

### PEX

```cpp
void set_pex_enabled(bool enabled);
```

Enables or disables Peer Exchange for this download. PEX cannot be enabled when the torrent is marked private (`DownloadInfo::is_private()` returns `true`).

### Upload and download slot limits

```cpp
uint32_t uploads_max() const;
void     set_uploads_max(uint32_t v);

void     set_uploads_min(uint32_t v);

uint32_t downloads_max() const;
void     set_downloads_max(uint32_t v);

uint32_t downloads_min() const;
void     set_downloads_min(uint32_t v);
```

| Method | Description |
|---|---|
| `uploads_max() / set_uploads_max(v)` | Get or set the maximum number of simultaneous upload slots for this download. |
| `uploads_min() / set_uploads_min(v)` | Get or set the minimum number of upload slots the resource manager will try to maintain. |
| `downloads_max() / set_downloads_max(v)` | Get or set the maximum number of simultaneous download slots (unchoked remote peers). |
| `downloads_min() / set_downloads_min(v)` | Get or set the minimum number of download slots. |

### Per-download throttles

```cpp
void set_upload_throttle(Throttle* t);
void set_download_throttle(Throttle* t);
```

Assigns a `Throttle` object to this download, overriding the global throttle for upload or download traffic respectively. Pass `nullptr` to revert to the global throttle.

## Connection type

```cpp
enum ConnectionType {
  CONNECTION_LEECH,
  CONNECTION_SEED,
  CONNECTION_INITIAL_SEED,
  CONNECTION_METADATA,
};

ConnectionType connection_type() const;
void           set_connection_type(ConnectionType t);
```

The connection type controls which peer connection implementation the download uses.

| Value | Description |
|---|---|
| `CONNECTION_LEECH` | Standard leeching mode. Downloads pieces from peers while also uploading. |
| `CONNECTION_SEED` | Seed-only mode. Serves pieces to peers without downloading. |
| `CONNECTION_INITIAL_SEED` | Initial seeding (super-seeding) mode. Strategically offers rarer pieces to maximize spread. |
| `CONNECTION_METADATA` | Metadata-only mode, used for magnet link resolution before the full `.torrent` info is available. |

## Choke heuristics

```cpp
enum HeuristicType {
  // No named values — values are implementation-defined
};

HeuristicType upload_choke_heuristic() const;
void          set_upload_choke_heuristic(HeuristicType t);

HeuristicType download_choke_heuristic() const;
void          set_download_choke_heuristic(HeuristicType t);

void update_priorities();
```

| Method | Description |
|---|---|
| `upload_choke_heuristic() const` | Returns the algorithm used to decide which peers to unchoke for uploading. |
| `set_upload_choke_heuristic(HeuristicType t)` | Sets the upload unchoke algorithm. |
| `download_choke_heuristic() const` | Returns the algorithm used to decide which peers to unchoke for downloading. |
| `set_download_choke_heuristic(HeuristicType t)` | Sets the download unchoke algorithm. |
| `update_priorities()` | Re-evaluates all peer bitfields to determine interest after you have changed file or piece priorities. This is moderately expensive; avoid calling it in tight loops. |

<Note>
  `HeuristicType` is declared as an empty enum in the current public API. Available values are implementation-defined; consult the source or configuration layer of your application for valid options.
</Note>

## Tracker control

```cpp
void send_completed();
void manual_request(bool force);
void manual_cancel();
```

| Method | Description |
|---|---|
| `send_completed()` | Sends a `completed` event to all trackers, signaling that the download has finished. The library calls this automatically when the last piece is verified; you only need to call it manually in edge cases. |
| `manual_request(bool force)` | Triggers an immediate tracker announce. When `force` is `true`, the request bypasses the minimum re-announce interval. |
| `manual_cancel()` | Cancels any pending manual tracker request previously scheduled with `manual_request()`. |
