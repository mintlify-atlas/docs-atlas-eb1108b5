libTorrent is a C++ BitTorrent library built for performance and correctness. It transfers data directly from memory-mapped file pages to the network stack, eliminating unnecessary copies. The library is designed to be embedded in torrent clients—[rTorrent](https://github.com/rakshasa/rtorrent) uses it as its core engine.

## What libTorrent provides

libTorrent gives you the full BitTorrent protocol stack as a linkable C++ library. Your application calls into the library to add downloads, manage peers, and respond to events—the library handles the rest.

<CardGroup cols={2}>
  <Card title="Zero-copy I/O" icon="bolt">
    Uses `mmap` to transfer file data directly to sockets without intermediate copies
  </Card>
  <Card title="DHT support" icon="network-wired">
    Full Kademlia DHT for trackerless peer discovery on the BitTorrent network
  </Card>
  <Card title="Bandwidth throttling" icon="gauge">
    Per-download and global upload/download throttles with per-address overrides
  </Card>
  <Card title="Protocol encryption" icon="lock">
    MSE/PE encryption with automatic plaintext fallback and retry logic
  </Card>
  <Card title="Resume data" icon="floppy-disk">
    Save and restore download state across restarts using built-in resume support
  </Card>
  <Card title="Multi-threaded" icon="microchip">
    Separate main, network, and tracker threads for non-blocking operation
  </Card>
</CardGroup>

## Get started

<Steps>
  <Step title="Build the library">
    Install build dependencies, run `autoreconf -ivf`, then `./configure && make && make install`. See [Building libTorrent](/building) for details.
  </Step>
  <Step title="Initialize in your application">
    Call `torrent::initialize_main_thread()` followed by `torrent::initialize()` at startup. These set up the thread infrastructure and internal managers.
  </Step>
  <Step title="Add a download">
    Parse a `.torrent` file into a `torrent::Object`, then call `torrent::download_add()` to create a `Download` handle. Call `open()`, then `hash_check()`, then `start()`.
  </Step>
  <Step title="Run the event loop">
    Drive the library by calling `torrent::this_thread::poll()->do_poll(timeout_usec)` in your main loop. The library calls your registered callbacks as events occur.
  </Step>
</Steps>

<Note>
  libTorrent is a library, not a standalone application. You embed it in your own C++ program. [rTorrent](https://github.com/rakshasa/rtorrent) is the reference client implementation.
</Note>
